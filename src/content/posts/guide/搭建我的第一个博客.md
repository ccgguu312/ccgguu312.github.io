# 从零开始建立并部署 my blog


## 1、环境准备

- 创建一个文件夹用来存放博客项目$\quad$------后续就以MyBlog命名文件夹

- 进入到MyBlog文件夹，打开cmd命令提示符，使用`xxx -v`检查以下工具是否安装以及版本号：
[git](https://git-scm.com/) $\qquad$ $\qquad$ $\qquad$ [Node.js](https://nodejs.org/zh-cn) $\qquad $ $\qquad$ $\qquad$ [npm](https://www.npmjs.com/)

-  配置npm镜像源,并下载 pnpm

```
npm config set registry https://registry.npmmirror.com

npm install -g pnpm
```

## 2、开始搭建

**Astro**是一种静态站点生成器，加载轻量，专门为**博客**等内容网址设计。

> 1 选取主题

- 可从[Astro](https://astro.build/themes/)官方选择喜欢的主题，点进去之后选择**Get Stared**进入到作者**github**仓库。
- 或是直接在github中寻找好看的博客仓库 $\quad$------我推荐https://github.com/saicaca/fuwari

> 2 克隆项目

- 进入到MyBlog文件夹，打开cmd输入：
```
git clone <喜欢的主题的github仓库地址>

#仓库地址可在仓库右上角code中HTTPS中复制
```
- 克隆好之后，还需要读取项目根目录下`package.json`的依赖并安装所声明的包：
```
cd<复制的项目文件夹名称>

pnpm install
```

> 3 本地与github关联

- 虽然克隆的项目本身存在`.git`，但此时本地项目还和github上博客项目关联，一般建议重新初始化git（当然跳过这一步也无伤大雅）
```
Remove-Item -Recurse -Force .git
git init
git add .
git commit -m "Initialize my new blog theme"
git branch -M main
```
- 在github中创建一个自己的仓库用来接收本地项目，需要注意的是，个人博客的项目名必须和用户名一致且要有`.github.io`的后缀，而项目博客命名则无限制。
  
![doocs](https://img2.nloln.de/file/BQACAgUAAyEGAASLVN5eAAJjCWpeFt-71KwsBqkNNMBXQuUmf2YmAALZHgACl9zxVgS9ZAyCm575PQQ.png)

- 创建好之后在本地博客项目文件夹`MyBlog/<博客项目名>`打开cmd输入：
```
git remote add origin <github仓库地址>
git push -u origin main
```
> 4 本地预览

- 写文章可用`pnpm new-post <文件名称>`生成**markdown**文件，或者也可以去原作者仓库找READNE说明，确定文章存放目录后手动在目录里创建markdown文件。

- 一些主题也可以找`config.yml`等配置文件来修改。
  
- 使用`pnpm dev`可在本地预览博客的内容，复制cmd给的网址打开即可查看。

![doocs](https://img2.nloln.de/file/BQACAgUAAyEGAASLVN5eAAJjCmpeGNlUw_5BCZduzPAPf1J5bckiAALfHgACl9zxVsgPUzhi7PBQPQQ.png)

## 3、部署到网页

**GitHub Pages**是一个免费的静态服务器，把本地生成的博客放到网页，可供其他人访问。

- 在clone的博客项目中查找是否有`.github\workflows\deploy.yml`一类的文件，这是**GitHub Actions**的配饰文件，里面包含`触发事件，执行的任务，任务步骤`等声明，GitHub Actions自动化工作流构建好博客之后即可交由GitHub Pages托管。

- 要实现Action与Pages协同，需要先在仓库打开`setting`，选择左侧Pages，在界面中找到`Build and deployment`，将其切换到`GitHub Action`

![doocs](https://img2.nloln.de/file/BQACAgUAAyEGAASLVN5eAAJjFmpeJ30yUnOw0-KNBbIyDng-FkwdAAISHwACl9zxVk4JInU9tfWQPQQ.png)

- 若未找到yml文件，则需在`<本地博客项目>.github\workflows\`路径下新建`deploy.yml`文件，编写内容：
```
name: Deploy to GitHub Pages

on:
  push:
    branches: [main] # 每次向 main 分支推送代码时自动触发

# 给这个 Action 授予修改 GitHub Pages 的权限
permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: 'pages'
  cancel-in-progress: true

jobs:
  build:
    name: 构建博客
    runs-on: ubuntu-latest
    steps:
      - name: Checkout your repository
        uses: actions/checkout@v4

      - name: Install pnpm
        uses: pnpm/action-setup@v4
        with:

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --no-frozen-lockfile

      - name: Build Astro
        run: pnpm run build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    name: 部署
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

- 最后还需要修改博客根目录下的 astro.config.mjs 文件，将域名替换为自己用户名下的博客仓库地址，如：
```yaml
// https://astro.build/config
export default defineConfig({
	site: "https://ccgguu312.github.io",

	base: "/",
    ············
    ······
    ···
```
- 这样每次推送项目时，都能在仓库的`Action`里获取博客的网址。

## 4、更新博客
- 在本地修改好文章或配置主题之后将本地项目推送到github仓库：
```yaml
git add .
git commit -m “f”
git push
```
